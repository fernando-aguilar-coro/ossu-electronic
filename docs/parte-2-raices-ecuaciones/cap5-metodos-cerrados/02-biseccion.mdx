# Método de Bisección

## Idea fundamental

El **método de bisección** divide el intervalo $[a, b]$ en dos mitades y selecciona la subparte donde la función cambia de signo. El proceso se repite hasta que el intervalo sea suficientemente pequeño.

Es el método cerrado más **simple y robusto**, aunque no el más eficiente.

---

## Algoritmo

Dado un intervalo inicial $[x_l, x_u]$ con $f(x_l) \cdot f(x_u) < 0$:

### Paso 1 — Calcular el punto medio

$$
x_r = \frac{x_l + x_u}{2}
$$

### Paso 2 — Evaluar y decidir

Calcular $f(x_r)$ y determinar el subintervalo:

$$
\text{Si } f(x_l) \cdot f(x_r) < 0 \implies \text{la raíz está en } [x_l,\, x_r] \implies x_u = x_r
$$

$$
\text{Si } f(x_l) \cdot f(x_r) > 0 \implies \text{la raíz está en } [x_r,\, x_u] \implies x_l = x_r
$$

$$
\text{Si } f(x_l) \cdot f(x_r) = 0 \implies x_r \text{ es la raíz exacta}
$$

### Paso 3 — Calcular error y repetir

$$
\varepsilon_a = \left| \frac{x_r^{(nuevo)} - x_r^{(anterior)}}{x_r^{(nuevo)}} \right| \times 100\%
$$

Repetir desde el Paso 1 hasta que $\varepsilon_a < \varepsilon_s$ o se alcance el número máximo de iteraciones.

---

## Análisis de convergencia

### Error verdadero máximo

Después de $n$ iteraciones, el ancho del intervalo se reduce a la mitad en cada paso:

$$
\Delta x_n = \frac{b - a}{2^n}
$$

El **error máximo absoluto** en la iteración $n$ es:

$$
E_n \leq \frac{b - a}{2^n}
$$

### Número de iteraciones necesarias

Para garantizar una tolerancia $\varepsilon$ en el resultado:

$$
n \geq \frac{\ln\!\left(\dfrac{b-a}{\varepsilon}\right)}{\ln 2} = \log_2\!\left(\frac{b-a}{\varepsilon}\right)
$$

### Tasa de convergencia

La bisección tiene convergencia **lineal** con tasa $r = 0.5$:

$$
E_{n+1} \approx \frac{1}{2} E_n
$$

## Pseudocódigo

```
función biseccion(f, xl, xu, es, imax):
    xr_anterior = xl
    iteracion = 0
    
    repetir:
        xr = (xl + xu) / 2
        iteracion = iteracion + 1
        
        si xr ≠ 0:
            ea = |( xr - xr_anterior) / xr| × 100
        
        producto = f(xl) × f(xr)
        
        si producto < 0:
            xu = xr
        si producto > 0:
            xl = xr
        si producto = 0:
            ea = 0
        
        xr_anterior = xr
    
    mientras ea > es  Y  iteracion < imax
    
    retornar xr
```

## Implementación del Algoritmo

```python
def biseccion(f, xl, xu, es=0.0001, imax=100):
    """
    Método de bisección para encontrar raíces.

    Parámetros
    ----------
    f    : función continua
    xl   : límite inferior del intervalo
    xu   : límite superior del intervalo
    es   : tolerancia de error aproximado (%)
    imax : número máximo de iteraciones
    
    Retorna
    -------
    xr   : aproximación de la raíz
    """
    if f(xl) * f(xu) > 0:
        raise ValueError("No hay cambio de signo en el intervalo dado.")

    xr_ant = xl
    ea = 100.0

    for i in range(1, imax + 1):
        xr = (xl + xu) / 2

        if xr != 0:
            ea = abs((xr - xr_ant) / xr) * 100

        producto = f(xl) * f(xr)

        if producto < 0:
            xu = xr
        elif producto > 0:
            xl = xr
        else:
            ea = 0

        xr_ant = xr
        print(f"  Iter {i:3d}: xl={xl:.6f}, xu={xu:.6f}, xr={xr:.6f}, ea={ea:.4f}%")

        if ea < es:
            break

    return xr


# --- Ejemplo de uso ---
import math

f = lambda x: math.exp(-x) - x   # raíz ≈ 0.5671

raiz = biseccion(f, xl=0, xu=1, es=0.5)
print(f"\nRaíz aproximada: {raiz:.6f}")
```

## Consideraciones prácticas

:::tip Elección del intervalo
Graficar $f(x)$ antes de aplicar bisección para identificar intervalos con cambio de signo y evitar raíces múltiples dentro del mismo intervalo.
:::

:::warning Raíces múltiples
Si $f(x)$ toca el eje $x$ sin cruzarlo (raíz de multiplicidad par), el producto $f(x_l) \cdot f(x_u) > 0$ y el método **no detecta** la raíz. En este caso, conviene analizar $f'(x)$ o refinar la búsqueda del intervalo.
:::

:::note Eficiencia
La bisección siempre converge pero es lenta. Para funciones suaves y bien comportadas, los métodos abiertos (Newton-Raphson) son considerablemente más rápidos.
:::
